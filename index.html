Introduction

LGBTQ+ representation in television and streaming media has become increasingly visible in recent years. Television series not only reflect social developments but also influence public perception of gender and sexuality. Because of this, analyzing LGBTQ+ representation in media is relevant for understanding broader cultural trends.

This project investigates LGBTQ+ characters in dramatic television series from the 2020s. The project combines automated data extraction, data cleaning, keyword-based label detection, and manual review in order to build a structured dataset of LGBTQ+ television characters.

The project also demonstrates how computational methods and reproducible data pipelines can be used for media and gender studies research.

Research Question

The central research question of this project is:

Which LGBTQ+ identities are represented most frequently in dramatic television series of the 2020s?

Additional questions include:

  • Which identities appear most often?
  • Which identities are underrepresented?
  • How effective is automatic identity classification using text-based methods?

Data Cleaning & Data Enrichment

Before the analysis could begin, the dataset needed to be cleaned and standardized.

The raw Wikipedia table contained formatting inconsistencies, citation markers, and additional text elements that were not useful for the analysis.

The following preprocessing steps were performed:

  • Removal of Wikipedia citation markers such as [1]
  • Removal of bracketed information such as (season 2)
  • Cleaning and normalization of character names
  • Standardization of text formatting
  • Removal of unnecessary whitespace

For example: 

TK Strand[1] (season 2)

became:

Code anzeigen 
import re

name = "TK Strand[1] (season 2)"

name = re.sub(r"\[.*?\]", "", name)
name = re.sub(r"\(.*?\)", "", name)

print(name)
TK Strand

After cleaning the dataset, the next step involved enriching the data with LGBTQ+ identity labels.

The project used a rule-based text classification approach implemented in Python.

The script analyzed the content of the Notes column and searched for keywords and relationship descriptions associated with LGBTQ+ identities.

Regular expressions (regex) were used to detect patterns such as:

  • gay
  • lesbian
  • bisexual
  • transgender
  • queer
  • non-binary
  • pansexual
  • asexual

Identity Detection

After cleaning and preparing the dataset, the next step involved automatically identifying LGBTQ+ identities.

The project used a rule-based classification system implemented in Python. Character descriptions contained in the Notes column were analyzed using keyword matching and regular expressions (regex).

The system searched for explicit references to LGBTQ+ identities such as:

  • Gay
  • Lesbian
  • Bisexual
  • Transgender
  • Non-binary
  • Pansexual
  • Asexual

For example, descriptions containing terms such as:

The character is gay.
The character is bisexual.
The character identifies as non-binary.

could be automatically assigned to the corresponding category.

Code anzeigen 
import re

text = "The character is bisexual."

if re.search(r"\bbisexual\b", text.lower()):
    print("Bisexual")
Bisexual
The detection process was implemented through predefined keyword patterns and regular expressions. This approach allowed a large portion of the dataset to be classified automatically before manual review became necessary.

The automatic classification stage significantly reduced the amount of manual work required during the later phases of the project.

Relationship Inference

Not all LGBTQ+ identities were explicitly stated in the character descriptions.

To improve the classification process, an additional relationship inference step was implemented.

The system analyzed relationship-related phrases such as:

Rachel and Dana are in a romantic relationship.

Character A and Character B are lovers.

Character X is dating Character Y.

Using the detected names and gender information, the script inferred potential LGBTQ+ identities based on the relationship context.

Code anzeigen 
character_gender = "female"
partner_genders = ["female"]

if character_gender == "female" and "female" in partner_genders:
    print("Lesbian")
Lesbian

For example, when two female characters were identified as being romantically involved, the system could infer a Lesbian label. Similarly, relationships between two male characters could contribute to a Gay classification.

This additional inference step increased the number of automatically classified characters and reduced the number of entries that required manual review.

The relationship inference process was particularly useful for descriptions that focused on relationships rather than explicitly mentioning sexual orientation or gender identity.

Manual Review

Despite the automated classification process, some character descriptions remained ambiguous or lacked sufficient information for reliable labeling.

When the system could not confidently assign a label, the character was marked as:

Code anzeigen 
found_labels = []

if not found_labels:
    print("[MANUAL_REVIEW_REQUIRED]")
[MANUAL_REVIEW_REQUIRED]

These entries were exported into a separate review file and manually inspected.

During this step, additional information from the original source material was used to assign the most appropriate LGBTQ+ identity label.

The manual review stage served as a quality assurance mechanism and helped improve the overall accuracy of the dataset.

After the review process was completed, the manually labeled records were merged with the automatically classified entries to create the final dataset used for analysis and visualization.

This combination of automated processing and manual validation resulted in a more complete and reliable dataset.

Key Findings

The analysis revealed several important patterns regarding LGBTQ+ representation in dramatic television series of the 2020s.

Three key findings emerged from the dataset and visualizations.

  1. Gay and Lesbian Representation Dominates

The majority of characters identified in the dataset were classified as either Gay or Lesbian. These two categories account for the largest share of LGBTQ+ representation across the analyzed television series.

This suggests that television productions continue to focus primarily on identities that have historically received the greatest visibility within mainstream media.

  1. Representation Increased Throughout the 2020s

The temporal visualizations indicate a general increase in the number of LGBTQ+ characters represented over the years.

Streaming platforms and international productions appear to contribute significantly to this development, resulting in greater LGBTQ+ visibility in contemporary television.

  1. Uneven Representation Across Identity Groups

While Gay and Lesbian identities were strongly represented, other LGBTQ+ identities appeared considerably less frequently.

Categories such as Non-binary, Intersex, Genderfluid, and Asexual accounted for only a small proportion of the dataset.

This suggests that LGBTQ+ representation has expanded, but diversity within LGBTQ+ representation remains uneven.

Overall, the findings indicate growing LGBTQ+ visibility in television while also highlighting continuing disparities between different identity groups.

Interpretation of Results

The visualization shows that gay and lesbian characters represent the largest proportion of LGBTQ+ identities within the dataset.

Gay and lesbian identities account for the majority of LGBTQ+ character representations in dramatic television series of the 2020s.

This suggests that television series continue to focus primarily on gay and lesbian representation, while other LGBTQ+ identities appear less frequently.

Identities such as:

  • Non-binary
  • Intersex
  • Genderfluid
  • Asexual

appear significantly less often in comparison.

One possible explanation is that mainstream television productions still rely more heavily on identities that are already more socially visible and widely represented in media culture.

The dataset also indicates that streaming platforms such as Netflix, HBO, Hulu, and Amazon Prime Video contribute strongly to LGBTQ+ representation in television series.

Many of the analyzed series originated from streaming services rather than traditional television networks.

Another important observation is that several characters were associated with multiple identity categories, demonstrating that LGBTQ+ identities are often complex and cannot always be reduced to a single label.

Overall, the analysis suggests that LGBTQ+ visibility in television has increased during the 2020s, but representation remains uneven across different identity groups.

Methodological Reflection

This project used a rule-based classification approach instead of machine learning methods.

The primary goal was to ensure transparency, reproducibility, and interpretability throughout the enrichment process.

The enrichment process relied primarily on:

  • keyword matching
  • regular expressions (regex)
  • relationship inference
  • manual validation

One advantage of this approach is that the classification process can be reproduced consistently across the dataset.

In addition, the method does not require large training datasets or complex machine learning models.

However, the approach also has several weaknesses.

Keyword-based systems depend heavily on the wording used in the Notes column. If identities or relationships are described indirectly, the system may fail to classify the character correctly.

For example, descriptions such as: 


"They are romantically involved."

provide limited contextual information and may require manual interpretation.

Limitations

Dependence on Wikipedia

The dataset was based entirely on information collected from Wikipedia.

Because Wikipedia is community-generated, the data may contain inconsistencies, incomplete entries, or outdated information.

Some television characters also had very limited descriptions, which made automatic classification more difficult.

Ambiguous Character Descriptions

Many entries in the Notes column described relationships indirectly without explicitly mentioning LGBTQ+ identities.

For example, descriptions such as: 


"X and Y are in a relationship."

provide insufficient information for reliable automatic classification and often require manual review.

Reproducibility

One of the main objectives of this project was to ensure that the entire workflow could be reproduced by other researchers.

The project follows a structured data pipeline that transforms raw Wikipedia data into a cleaned, enriched, and analyzed dataset. All processing steps were implemented in Python and documented within the GitHub repository.

The workflow can be reproduced by executing the scripts in the following order:

  1. data_extraction.py
    • Extracts the original table from Wikipedia and stores it as a CSV file.
  2. data_enrichment.py
    • Performs the initial LGBTQ+ identity detection based on predefined keyword patterns.
  3. data_enrichment_plus.py
    • Applies additional enrichment methods, including relationship inference and gender-based classification.
  4. manual_review_needed.csv
    • Contains entries that could not be classified automatically and therefore required manual validation.
  5. merge_final_data.py
    • Combines automatically classified records with manually reviewed entries to create the final dataset.
  6. advanced_visualizations.py
    • Generates the visualizations used throughout the analysis.
  7. report.qmd
    • Produces the final report using Quarto.

All required Python packages are listed in the requirements.txt file, allowing the computational environment to be recreated easily.

The final dataset used for analysis is stored as:

data/processed/final_labeled_data.csv

The complete project is managed using Git and GitHub, which enables version control, collaboration, and transparency throughout the development process.

By combining Python scripts, structured datasets, version control through GitHub, and Quarto reporting, the project provides a transparent and reproducible workflow from data collection to final analysis.
